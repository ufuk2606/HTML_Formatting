Semantics in HTML

👉 In HTML, for example, the <h1> element is a semantic element, which gives the text it wraps around the role (or meaning) of " a top level heading on your page."

Here is a top level heading

👉 By default, most browsers user agent stylesheet will style <h1> with a large font size to make it look like a heading (although you could style it to look anything you wanted) On the other hand, you could make any elemnt look like a top level heading.

Consider the following

➮This will render it to look like a top level heading, but it has no semanticvalue, so it will not get any extra benefits as describes as above. It is therefore a good idea to use the right HTML element for the right job.

➮HTML should be coded to represent the data that will be populated and not based on its default presentation styling. Presentation (how it should look ), is the sole responsibility of CSS.

Some of the benefits from writing semantic markup are as follows:
  • Search engines will consider its content as importent keywords to influence the page`s search rankings (See SEO )
  • Screen readers can use it as a signpost to help visually impaired users navigate a page
  • Finding blocks of meainingful code is significantly easier than searching through endless divs with or without semantic or namespaced classes
  • Suggests to the devoloper the type of data that will be populated
  • Semantic neaming mirrors proper custom element/component naming
